Getting started with dmcs

Before utilizing DMCS, you need to properly set up the system.

Step 1: If you decide to use a manager for your system, then proceed with this step, otherwise skip to the next step.

$ dmcsm [--port=PORT]

The command-line argument --port informs the daemon the port that it is going to use for communication.

Step 2: For each context in the system run the following command on its designated machine.

$ dmcsd --context=NUM --kb=C_NUM.lp --br=C_NUM.br --manager=HOST:PORT

Command-line arguments:

  • --context: identifier of the context that the daemon represents,
  • --kb: name of the file that stores the local knowledge base,
  • --br: name of the file that stores bridge rules,
  • --manager: set up the location of the manager.

Step 3: The equilibria of the system wrt. context ROOT can be computed by running:

$ dmcsc --context=ROOT --manager=HOST:PORT

The paramters have the same meaning as above.

Running the developing version of dmcs

Without the present of the manager, one can run the system as follows.

Step 1: Setup the daemons for all contexts in the system

$ dmcsd --context=NUM --port=PORT --kb=C_NUM.lp --br=C_NUM.br --topology=FILENAME

where the --topology argument specifies the file which sores the pre-computed topology of the system. There are two options: either the original topology in a .top file or the optimal one in a .opt file. Note that all contexts must agree on the type of topology loaded.

Step 2: Computing the equilibria now depends on what kind of topology was loaded in the previous step:

  • If the original topology was loaded, run: 
    ./dmcsc --hostname=HOST --port=PORT --system-size=NUM --query-variables=QUERY_VARS
  • Otherwise, when the optimal topology was loaded, run: 
    ./dmcsc --hostname=HOST --port=PORT --system-size=NUM

where:

  • --hostname and --port hold hostname and port of the ROOT context to be queried,
  • --system-size is the number of context in the MCS, and
  • --query-variables specifies a list of intergers (encompassed in a string), each is corresponding to a filter of atoms for the output of a context in the system, in which bit position i of the k-th interger in the list is set to 1 if the user wants to see the truth value of the i-th atom in context Ck. Note that in our test cases, atoms in a context Ck are ordered as ak, bk, ck,... i.e., ak has position 1, bk has position 2, and so on.

In all of our test cases, the above command lines are automatically generated. More conveniently, we provide shell scripts that automatically execute all the setup and query steps in just one run. For this, run (note that we need all test cases stored in a sub directory 'test' of the current directory): 

$ /bin/bash tests/SHELL_SCRIPT_NAME
