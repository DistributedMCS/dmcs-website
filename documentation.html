Getting started with dmcs

Before you can utilise DMCS, you have to properly set up the system.
Step one:
If you decide to use a manager for your system, then proceed with this step, otherwise skip to the next step.

$ dmcsm [--port=PORT]

The command-line argument --port informs the daemon the port that it is going to use for communication.
Step Two:
For each context in the system run the following command on its designated machine.

$ dmcsd --context=NUM --kb=C_NUM.kb --br=C_NUM.br --manager=HOST:PORT

The command-line argument --context tells the daemon the context id that it will represent. The knowledge base and the bridge rule files are provided via --kb and --br, respectively. The --manager option is used to set up the location of the manager component.
If you have decided to run the system without the manager, then simply omit the --manager option.
Step Three:
Now that everything is set up, you can compute the equilibria of M w.r.t context ROOT as follows:

$ dmcsc --context=ROOT --manager=HOST:PORT

The paramters have the same meaning as above.
If you have decided to run the system without the manager, then simple replace the --manager option with --config=CONFIG_FILE, where CONFIG_FILE contains all the host names and ports of all contexts in the system,click here for an example.
